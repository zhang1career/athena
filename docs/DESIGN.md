# Athena 预测平台 — 设计文档

## 1. 总体架构

Athena 采用**平台 + 应用**的分层架构：平台负责通用流程与抽象，应用负责领域数据与配置。二者通过清晰接口协作。

│                        Applications                              │

│  │ - 数据源            │  │ - 数据源             │               │
│  │ - 配置              │  │ - 配置               │               │
│  │ - 专有工具          │  │ - 专有工具           │               │

## 2. 核心设计理念

### 2.1 策略即插件

- 所有策略实现统一的 `Strategy` 接口
- 通过插件注册机制动态加载
- 组合策略 (`MetaStrategy`) 与基础策略同构，可递归组合

### 2.2 实验驱动

- 一切预测行为均通过「实验」执行
- 实验记录完整上下文：参数、数据切分、指标、血缘

### 2.3 可观测优先

- 预测过程可追踪：输入 → 中间结果 → 输出
- 支持日志、指标、特征的可检索与导出

### 2.4 AI 可操作

- 参数配置化、API 结构化
- AI 可读、可写策略定义与实验配置

## 3. 策略体系

### 3.1 策略层次

```
BaseStrategy（基类）
    ├── SingleStrategy（单一预测模型）
    │       ├── SklearnStrategy
    │       ├── LightGBMStrategy
    │       └── ...
    └── MetaStrategy（组合策略）
            ├── WeightedEnsemble（加权平均）
            ├── StackingStrategy（Meta Model）
            └── CustomCombiner（可编程组合）
```

### 3.2 策略接口契约

```python
# 概念示意
class Strategy(ABC):
    def fit(self, X, y, **kwargs) -> None: ...
    def predict(self, X, **kwargs) -> PredictResult: ...
    def get_params(self) -> dict: ...
    def set_params(self, **params) -> None: ...
    @classmethod
    def get_schema(cls) -> StrategySchema: ...  # 参数 schema，供 AI 发现
```

### 3.3 组合策略（Meta Model）

- **WeightedEnsemble**：对多个子策略预测结果加权平均
- **Stacking**：用元模型学习子策略输出的组合权重
- **自定义组合**：允许 AI 编写组合逻辑（如条件分支、规则）

## 4. 实验框架

### 4.1 实验生命周期

```
创建实验 → 加载策略与配置 → 数据加载 → 切分（train/val/test）
    → 训练/调参 → 回测 → 评估 → 持久化 → 完成
```

### 4.2 与回测的集成

- 回测是实验的一个阶段，而非独立流程
- 时间序列按 Walk-Forward 或指定方式切分
- 每次预测及中间状态可记录，便于事后分析

### 4.3 分布式支持

- 实验单元可提交到 Ray 集群
- 参数搜索的 trial 可并行执行
- 平台提供统一的「运行器」抽象，支持本地与分布式

## 5. AI Research Loop

```
┌──────────────────┐
│ AI propose       │  生成/修改策略、参数、组合
│ strategy         │
└────────┬─────────┘
         ▼
┌──────────────────┐
│ System run       │  实验框架执行
│ experiment       │
└────────┬─────────┘
         ▼
┌──────────────────┐
│ Evaluate         │  计算指标、写入 DB
│ metrics          │
└────────┬─────────┘
         ▼
┌──────────────────┐
│ AI analyze       │  读取结果、对比、诊断
│ results          │
└────────┬─────────┘
         ▼
┌──────────────────┐
│ AI modify        │  调整策略/参数，进入下一轮
│ strategy         │
└────────┬─────────┘
         │
         └──────────────────────────────────┐
                                            ▼
```

- 每一轮循环对应一次或一批实验
- AI 通过 API 读取指标、实验元数据，决定如何修改
- 平台不强制 AI 的实现方式（脚本、Agent、定时任务均可）

## 10. 风险与缓解

| 风险 | 缓解 |
|------|------|
| 策略接口演化导致兼容问题 | 版本化 schema，向后兼容策略 |
| 实验量过大导致存储压力 | 归档策略、分层存储 |
| AI 生成的策略存在安全隐患 | 沙箱执行、代码审查策略 |
| 分布式故障恢复 | 实验幂等、断点续跑设计 |
