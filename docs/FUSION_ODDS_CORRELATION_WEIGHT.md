# 融合架构下「博彩赔率」的中间结果与相关度权重

## 1. 运算架构回顾

项目的运算架构可以概括为：

```
来源数据 → 多个中间结果 → 加权融合 / Meta Model（Stacking）→ 最终结果
```

- **来源数据**：原始数据（如比赛记录、赔率表、ELO 等）。
- **中间结果**：各分支对同一预测目标产出的结果，例如：
  - 博彩赔率隐含概率（出线/胜平负等）；
  - ELO 或其它实力模型给出的分数/概率；
  - LightGBM 等模型在对应任务上的预测。
- **融合**：对多个中间结果做**加权**（`WeightedEnsemble`）或 **Stacking**（元模型学习组合），得到最终预测。

博彩赔率作为其中一个**中间结果**，参与融合时需要一个合理的权重（或权重来源）。下面说明：对赔率做**单独的、具有相关度含义的融合计算**，得到一组参数，并把这组参数视为（或推导出）融合时赔率的加权值，是合理且可落地的。

---

## 2. 赔率作为「中间结果」的单独融合计算

### 2.1 思路

- **不**用赔率去训练一个「从赔率到出线」的判别模型（见 [GROUP_WINNER_ODDS_ANALYSIS.md](./GROUP_WINNER_ODDS_ANALYSIS.md)）。
- **而是**：在历史数据上，把赔率视为**现成的概率估计**，只做**相关性/校准评估**，得到一组**具有相关度含义的参数**。
- 这组参数可以：
  - **直接**作为融合时「赔率」这一路的权重（或权重的主要依据）；
  - 或作为 Stacking 时元模型的输入（例如「赔率可靠性」特征），由元模型间接决定赔率在组合中的占比。

即：**对博彩赔率做单独的融合计算 → 得到相关度含义的参数 → 用作（或推导）融合时赔率的加权值**。

### 2.2 可选的「相关度含义」参数

在历史数据上，用「赔率隐含概率」作为对 is_winner（或其它目标）的预测，计算下面一类或多类指标，作为**具有相关度含义的参数**：

| 参数 | 含义 | 与权重的关系 |
|------|------|--------------|
| **AUC** | 赔率作为二分类得分时，对正负类的排序能力 | 越高越可信，可映射为更大权重（如 w ∝ AUC - 0.5，或归一化到 [0,1]） |
| **Spearman / Kendall** | 组内或全局「赔率排序 vs 实际排名/结果」的相关系数 | 越高越可信，可映射为更大权重 |
| **Brier score** | 赔率概率与 0/1 结果的均方误差，越小越准 | 可转为「可靠性」再映射为权重（如 w ∝ 1 / (1 + Brier)） |
| **校准斜率/截距** | 隐含概率 vs 实际频率的线性拟合（或分段校准） | 斜率接近 1、截距接近 0 表示校准良好，可据此调权重或校准后再参与融合 |

可只取其一（如 AUC），也可取多个组成**参数向量**，再通过一个简单规则或一个小型元模型映射为**单一权重** \( w_{\text{odds}} \)：

- 例如：\( w_{\text{odds}} = \frac{\max(0, \text{AUC} - 0.5)}{0.5} \) 再与其它分支权重做归一化；
- 或：用 AUC、Brier、Spearman 等组成特征，训练一个轻量级「权重分配器」输出 \( w_{\text{odds}} \)。

### 2.3 在流水线中的位置

```
历史数据 (赔率 + 出线结果)
        ↓
  [赔率单独融合计算]
  输入：赔率隐含概率 + 真实 is_winner（及可选的分组/时间）
  输出：具有相关度含义的参数 θ（如 AUC, Brier, Spearman, 或向量）
        ↓
  [权重推导]
  w_odds = f(θ)  或由元模型根据 θ 与其它分支一起得到权重
        ↓
  [整体融合]
  final = w_odds * odds_score + w_elo * elo_score + ...  （加权）
  或    = meta_model(odds_score, elo_score, ...; θ 可作为特征)  （Stacking）
```

- **赔率分支的中间结果**：仍然是「赔率隐含概率」（或经上述校准后的概率），不改变现有数据流。
- **单独融合计算**：仅针对赔率 + 结果，产出 θ，不训练「预测出线」的模型。
- **融合权重**：\( w_{\text{odds}} \) 由 θ 得到，因此可以认为**融合时博彩赔率的加权值就是由这组具有相关度含义的参数所决定（或与之一一对应）**。

## 3. 与现有融合组件的对应关系

- **WeightedEnsemble**（`platform_core/strategy/meta.py`）：  
  `combiner_config.weights` 中对应「赔率」那一维的权重，可以**来自**上述单独融合计算得到的 \( w_{\text{odds}} \)（或由 θ 经 \( f(\theta) \) 得到）。这样，**这组参数就等价于（或直接推导出）融合时博彩赔率的加权值**。

- **StackingStrategy**：  
  元模型在组合各子策略输出时，可以把 θ（或 \( w_{\text{odds}} \)）作为额外特征或先验：例如「赔率在本届/本任务上的 AUC」，让元模型学习在多大程度上信任赔率，从而间接得到赔率在融合中的权重。

---

## 4. 小结

- **运算架构**：来源数据 → 多个中间结果 → 加权 / Stacking 融合 → 最终结果；**博彩赔率作为其中一个中间结果**是合理的。
- **单独融合计算**：仅针对「赔率 + 出线结果」做**相关度/校准评估**（AUC、Brier、Spearman、校准系数等），得到**具有相关度含义的一组参数 θ**，不训练「赔率→出线」的判别模型。
- **权重含义**：这组参数可以**直接或通过简单映射**作为融合时博彩赔率的加权值 \( w_{\text{odds}} \)；在 WeightedEnsemble 中对应赔率分支的权重，在 Stacking 中可作为元模型决定「赔率占比」的依据。

因此，**对博彩赔率做单独的、以相关度为含义的融合计算，得到一组参数，并把这组参数视为（或用于得到）融合时博彩赔率的加权值**，在概念上与现有融合架构一致，且可落地实现。

---

## 5. 实现与用法（代码）

- **赔率相关度 θ**：`platform_core.fusion.compute_odds_correlation(y_proba_positive, y_true)` → 返回 `{ "auc", "brier", "spearman", "suggested_weight" }`。
- **由 θ 推导融合权重**：`platform_core.fusion.compute_fusion_weights_with_odds_correlation(sub_predictions, y_true, odds_strategy_index)` → 返回归一化后的权重列表。
- **赔率基线策略**：`odds_baseline_group_winner`，不训练，`predict(X)` 用最后一列（或 `odds_column_index`）作为 P(小组第一)。
- **加权融合 + 自动权重**：使用策略 `weighted_ensemble`，参数示例：
  - `sub_strategies`: `["odds_baseline_group_winner", "lightgbm_match"]`（或其它支持该 task 的策略）
  - `combiner_config`: `{ "weight_estimator": "odds_correlation", "odds_strategy_index": 0 }`
  - 在 `fit(X, y)` 时会先解析子策略 ID、再训练子策略，然后根据赔率分支的预测与 `y` 计算 θ 并写入 `combiner_config["weights"]` 和 `combiner_config["odds_correlation_theta"]`。
