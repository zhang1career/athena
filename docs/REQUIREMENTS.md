# Athena 预测平台 — 需求文档

## 1. 项目概述

Athena 是一个基于 Python 的、AI 主导的研究型预测平台。平台提供统一的实验框架、策略抽象、回测与可观测能力，支持多种预测应用接入。
首个应用场景为**足球世界杯预测**。

---

## 2. 功能性需求

### 2.1 多策略与插件化

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-101 | 平台必须支持多策略架构，策略以插件形式注册与加载 | P0 |
| FR-102 | 支持策略的热插拔，无需重启即可加载新策略 | P1 |
| FR-103 | 策略必须具备统一的接口契约（输入/输出格式） | P0 |
| FR-104 | 策略的元数据（名称、版本、参数 schema）可被平台自动发现 | P0 |

### 2.2 策略组合（Meta Strategy）

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-201 | 支持策略组合，组合结果本身也作为策略使用 | P0 |
| FR-202 | 支持 Meta Model / Stacking 等组合方式 | P0 |
| FR-203 | 组合策略的权重逻辑可配置、可编程，AI 可修改 | P0 |
| FR-204 | AI 可生成新的组合策略或修改现有组合 | P1 |

### 2.3 实验框架

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-301 | 所有策略必须通过统一实验框架运行，禁止直接调用 | P0 |
| FR-302 | 实验框架支持：训练、验证、测试、回测阶段 | P0 |
| FR-303 | 支持多次实验的批量调度与排队 | P1 |
| FR-304 | 支持实验的取消、暂停、恢复 | P2 |

### 2.4 可观测性

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-401 | 预测过程必须完全可观测，包括中间预测值、特征、日志 | P0 |
| FR-402 | 实验的输入、输出、参数、指标可追溯 | P0 |
| FR-403 | 支持实验级别的日志聚合与检索 | P1 |

### 2.5 实验存储

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-501 | 所有实验必须持久化到数据库 | P0 |
| FR-502 | 实验记录包含：run_id、策略、参数、指标、时间戳、父实验等 | P0 |
| FR-503 | 支持实验的版本回溯与血缘分析 | P1 |

### 2.6 AI 友好设计

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-601 | 所有参数必须配置化，禁止硬编码 | P0 |
| FR-602 | 支持自动参数搜索：Bayesian optimization、Grid search | P0 |
| FR-603 | 自动记录每次调参实验，便于对比 | P0 |
| FR-604 | 提供结构化 API 供 AI Agent 调用（创建实验、查询结果、修改策略） | P0 |

### 2.7 回测系统

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-701 | 必须提供回测系统，避免过拟合 | P0 |
| FR-702 | 支持时间序列分割（如 Walk-Forward）、交叉验证 | P0 |
| FR-703 | 回测指标可配置，与业务应用解耦 | P1 |

### 2.8 可视化

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-801 | 提供简单 Dashboard，展示实验列表、指标对比 | P0 |
| FR-802 | Dashboard 信息对 AI 和人类均易理解 | P1 |
| FR-803 | 支持关键指标的趋势图、对比表 | P1 |

### 2.9 AI Research Loop

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-901 | 实现 AI 自动改进闭环：提议 → 实验 → 评估 → 分析 → 修改 | P0 |
| FR-902 | AI 可自动生成新策略或组合策略 | P1 |
| FR-903 | AI 可基于评估指标自动调整参数或权重 | P1 |

### 2.10 应用与平台分工

| 需求 ID | 描述 | 优先级 |
|---------|------|--------|
| FR-1001 | 平台提供：流程框架、通用工具、策略抽象、实验框架、回测、存储 | P0 |
| FR-1002 | 应用提供：数据源、配置文件、专有工具（如领域特征工程） | P0 |
| FR-1003 | 首个应用：足球世界杯预测 | P0 |

## 3. 非功能性需求

### 3.1 能力要求

| 需求 ID | 描述 |
|---------|------|
| NFR-101 | **实验记录 / 可观测**：每次实验可追溯、可复现 |
| NFR-102 | **参数调优**：支持多种搜索策略，结果可对比 |
| NFR-103 | **策略扩展**：新增策略不侵入平台核心 |
| NFR-104 | **分布式运行**：支持多机、多进程实验并行 |
| NFR-105 | **AI 可操作**：API 与数据格式对 AI Agent 友好 |

### 3.2 技术约束

| 需求 ID | 描述 |
|---------|------|
| NFR-201 | 核心语言：Python |
| NFR-202 | Web 框架：Django 4.1.13 |
| NFR-203 | 数据库：MySQL 5.7 |
| NFR-204 | 可引入：Hydra、Optuna、Ray、MLflow 等 |

### 3.3 性能与可用性

| 需求 ID | 描述 |
|---------|------|
| NFR-301 | 单次实验应在合理时间内完成（视数据量与策略而定） |
| NFR-302 | 实验队列应支持至少 100 个实验的排队 |
| NFR-303 | Dashboard 在常见数据量下响应时间 < 3s |

## 4. 术语表

| 术语 | 定义 |
|------|------|
| 策略 (Strategy) | 实现统一接口的预测模型或逻辑，可单独或组合使用 |
| 组合策略 (Meta Strategy) | 由多个基础策略按某种规则（如权重、 stacking）组合而成的策略 |
| 实验 (Experiment / Run) | 一次完整的训练/验证/测试或回测执行 |
| 回测 (Backtest) | 在历史数据上模拟预测，评估策略表现 |
| AI Research Loop | AI 自动提议、运行、评估、分析、修改的闭环流程 |
